> ## OperatingSystem

## 1.1 운영체제의 의미  
```
운영체제란 하드웨어 바로 위의 소프트웨어 계층으로    
컴퓨터 시스템을 더 편리하게 사용할 수 있는 환경을 만들고 컴퓨터 시스템의 자원을 효율적으로 관리하는 자원관리자를 의미한다   
   
협의의 운영체제(커널)는 부팅 이후 메모리에 상주하는 부분을 의미하며 광의의 운영체제는 커널과 각종 주변 시스템 유틸리티를 포함한 개념을 의미한다.   
보통 운영체제에 대해서 말할 때에는 커널로서의 운영체제를 말한다.   

```

## 1.2 운영체제의 목적    
> ### 1.2.1 자원을 효율적으로 관리   

```
1) 한정적으로 주어진 자원에 대해서 사용자간 형평성을 지키면서 최대한 효율적으로 자원을 사용할 수 있도록 관리하며
2) 사용자 및 OS 자기 자신을 보호하고
3) 프로세스, 파일, 에너지 등을 관리한다.
```

> ### 1.2.2 사용하기 편리한 환경 제공   

```
1) 여러 프로그램을 실행하거나 혹은 여러 사용자가 사용을 할 때도 마치 독자적으로 실행되고 있는 것 처럼 보이게 하고
2) 하드웨어를 직접적으로 다루는 복잡한 기능을 수행한다.
```

## 1.3 운영체제의 분류   
> ### 1.3.1 동시 작업 가능 여부   
>   > 단일 작업 운영체제
>   > ```
>   > 단일 작업만을 수행하며 하나의 작업이 끝나기 전까지 다른 작업은 하지 못하는 운영체제로 최근에는 보기 힘들며 MS-DOS 등의 운영체제가 있다. 
>   > ```   
>   > 다중 작업 운영체제
>   > ```
>   > 동시에 두 가지 이상의 작업을 처리하는 운영체제를 의마하며 대부분의 운영체제 (ex. UNIX, MS Windows 등)가 이에 속한다.
>   > ```   


> ### 1.3.2 사용자의 수   
>   > 단일 사용자 (single user)
>   > ```
>   > 단일 사용자만 접속할 수 있는 운영체제로 MS-DOS, 초기의 MS Windows 등이 있다. 
>   > ```   
>   > 다중 사용자 (multi user)
>   > ```
>   > 한대의 컴퓨터로도 다수의 사용자가 동시에 접속할 수 있는 운영체제로 단일 사용자 운영체제에 비해 접근성 관련 보안이 추가되고, 여러 사용자에 따른 형평성 있는 리소스 분배를
>   > 필요로 하는 운영체제이다. UNIX, NT server 등이 있다. 
>   > ```   


> ### 1.3.3 처리 방식   
>   > 일괄 처리 (batch processing)
>   > ```
>   > 요청된 작업을 일정 수준만큼 모아서 한번에 처리하는 운영체제로 다음 작업 처리시 이전 작업이 종료될 때 까지 기다려야 한다. 
>   > 최근에는 사용되지 않으며 Punch Card 처리 시스템 등이 있었다.  
>   > ```   
>   > 시분할 (time sharing)
>   > ```
>   > 여러 작업을 처리능력에 따라 일정시간 단위로 분할하여 사용하는 운영체제로 일괄 처리에 비해 상당히 짧은 응답시간을 가지고 있으며 interactive 한 방식을 가지고 있다. 
>   > 다만 실시간 처리방식과 달리 DeadLine 이 정해져 있지는 않다.
>   > ```   
>   > 실시간 (Realtime)
>   > ```
>   > DeadLine 이 있어 정해진 시간 내에 종료를 보장하는 실시간 시스템을 위한 운영체제로 DeadLine 이 지켜지지 않으면 심각한 오류를 초래할 수 있는 Hard Realtime System(경성 실시간 시스팀) 과 
>   > DeadLine이 정해져 있지만 지켜지지 않아도 크게 지장이 없는 Soft Realtime System(연성 실시간 시스템) 으로 나눠진다. 경성 실시간 시스템에는 원자로 관련 설비, 반도체 장비 등 파이프 라인으로 
>   > 구성되어 있는 시스템이나 미사일 제어 등의 시스템이 포함되고 연성 실시간 시스템에는 스트리밍 서비스 등이 포함된다.
>   > ```   

## 1.4 운영체제의 예   
### 1.4.1 유닉스   
```
유닉스는 대형 컴퓨터에 사용하기 위해 개발된 운영체제로 multitasking 과 multiuser 를 지향하며 개발되었다. 커널의 대부분이 C언어로 구성되어 있어 높은 이식성,확장성을 가지고 있다.
커널은 컴퓨터가 부팅되면 메모리상에 상주하는데 유닉스의 커널은 최소한의 구조를 가지고 있어 커널이 차지하는 메모리를 최소화 하였다.
초기에는 완전 OPEN SOURCE 였기 떄문에 다양한 종류의 운영체제 (ex. SunOs, Solaris 등) 가 만들어졌지만 이 중 Linux를 제외하면 현재는 완전 OPEN SOURCE 는 아니다. 
```   
### 1.4.2 MS Windows       

```
DOS 는 초기에 개인용 컴퓨터를 위해 개발된 운영체제이다. 그렇기 때문에 초기에는 single task, single user 를 지향하였다. 그렇기 때문에 DOS 에서 떨어져나온 MS Windows는 multitasking 을 구현하기는 했지만 처음에는 많이 불안정했다. 지금은 많이 안정화된 상태이며 MS 사의 다중 작업용 GUI 기반의 운영체제가 되었다. Windows 는 풍부한 자원 소프트웨어이며 Plug and Play 로 네트워크 환경을 강화하였다. 
```

## 1.5 운영체제의 구조   
> ### 1.5.1 CPU
> ```
> 운영체제는 CPU 의 스케줄링을 담당한다. 이는 단순히 시간 순, 용량 순으로 CPU 가 작업을 처리하는 것이 아니라 프로세스 별로 형평성을 고려하여 최선의 효율을 낼 수 있도록 CPU 의 작업 스케줄링을 한다. 
> ```
> 
> ### 1.5.2 MEMORY
> ```
> 모든 기기의 MEMORY 는 한정되어 있다. 그렇기 떄문에 운영체제는 한정된 메모리를 효율적으로 사용할 수 있도록 메모리를 분배해준다. 
> ```
> 
> 
> ### 1.5.3 Disk
> ```
> 운영체제는 Disk 의 파일관리(디스크 스케줄링) 도 한다. 디스크는 헤드가 직접 움직이기 때문에 운영체제는 Disk 가 최소한의 움직임으로 최대한 빠르게 처리할 수 있도록 스케줄링한다. 
> ```
> 
> 
> ### 1.5.4 I/O Device
> ```
> 운영체제는 입출력에 대해서 관리하는데 interrupt 방식을 사용한다고 한다. I/O 에서 CPU 에 입출력 관련 요청을 하면 CPU 에서 처리하는 방식이다. 
> ```
> 
> ### 1.5.5 그 외 
> ```
> * 프로세스의 생성 및 삭제
> * 자원 할당과 반환
> * 프로세스간 협력
> * 보호 시스템
> * 네트워킹
> * 명령어 해석 등의 역할로 구성되어 있다. 
> ```
